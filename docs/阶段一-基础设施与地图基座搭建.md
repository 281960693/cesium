# 阶段一：基础设施与地图基座搭建

## 1.1 前端工程初始化

### 技术选型原理

**为什么选择 Vue3 + Vite？**

| 特性 | Vue3 | React | 说明 |
|------|------|-------|------|
| 学习曲线 | 低 | 中 | Vue3 Composition API 更直观 |
| TypeScript 支持 | 优秀 | 优秀 | 两者都很好 |
| 生态系统 | 成熟 | 更大 | GIS 场景足够用 |
| 包体积 | 更小 | 更大 | 首屏加载更快 |

**为什么选择 Vite？**

```
传统 Webpack：
  启动：分析整个依赖图 → 编译所有模块 → 启动服务器 (30s+)
  热更新：编译受影响的模块链 (1-3s)

Vite：
  启动：预构建依赖 → 启动服务器 (<1s)
  热更新：精确匹配模块 (毫秒级)
```

Vite 使用原生 ES Module，开发体验极佳，特别适合 Cesium 这种大型库的开发调试。

### 工程化配置

```javascript
// vite.config.js 核心配置
{
  // 1. Cesium 静态资源处理
  assetsInclude: ['**/*.glb', '**/*.json'],

  // 2. Cesium Worker 路径配置
  define: {
    CESIUM_BASE_URL: JSON.stringify('/cesium/')
  },

  // 3. 开发代理 - 解决跨域
  server: {
    proxy: {
      '/api': {
        target: 'http://localhost:8001',
        changeOrigin: true
      }
    }
  }
}
```

## 1.2 Cesium 引擎集成

### Cesium.js 核心概念

```
Cesium 架构层级：
┌─────────────────────────────────────┐
│           Widget Layer             │  ← 地球控件、UI 组件
├─────────────────────────────────────┤
│           Scene Layer              │  ← 3D 场景、实体、图层
├─────────────────────────────────────┤
│         Primitive API              │  ← 底层渲染接口
├─────────────────────────────────────┤
│      WebGL / Web Workers           │  ← GPU 加速、多线程
└─────────────────────────────────────┘
```

### 地球初始化流程

```javascript
// 初始化步骤
1. 创建 Viewer 实例
   └→ 自动创建 CesiumWidget、Scene、Camera

2. 配置底图服务
   └→ 影像图层 (ImageryLayer)
   └→ 地形图层 (TerrainProvider)

3. 清理默认控件
   └→ 隐藏时间轴、动画控件
   └→ 简化导航控件

4. 配置相机初始位置
   └→ 设置经度、纬度、高度
   └→ 设置朝向 (heading, pitch, roll)
```

### 底图服务选择

```
天地图 (推荐免费方案)：
  - 影像底图：EPSG:4326 / EPSG:3857
  - 地形数据：全球 DEM
  - 限制：每日 10000 次调用

高德地图：
  - 影像底图：GCJ-02 坐标系
  - 注意：需要坐标转换

Mapbox：
  - 免费额度：50000 次/月
  - 优点：样式丰富
```

### 消息数据模型

```typescript
// 消息类型定义
interface Message {
  id: string
  role: 'user' | 'assistant' | 'system'
  content: string
  timestamp: number
  // 支持图文混排
  attachments?: Attachment[]
  // 工具调用结果
  toolCalls?: ToolCall[]
}

interface ToolCall {
  id: string
  name: string
  arguments: Record<string, any>
  result?: any
}
```

### 流式渲染原理

```
传统模式 (一次性返回)：
  User: "查询附近医院"
  ............. (等待 5s)
  Assistant: "查询到 10 家医院..."

SSE 流式模式 (边思考边输出)：
  User: "查询附近医院"
  Assistant: "我来帮你查询..."
  Assistant: "正在调用 POI 检索工具..."
  Assistant: "查询到 10 家医院..."
  Assistant: "已为你标注在地图上"

用户体验提升：感知等待时间减少 60%+
```

## 1.6 开发环境搭建指南

### 前置要求

```bash
# Node.js 版本
node -v  # >= 18.0.0

# npm 版本
npm -v   # >= 9.0.0

# 推荐使用 nvm 管理版本
nvm install 18
nvm use 18
```

### 启动命令

```bash
# 1. 安装前端依赖
cd frontend
npm install

# 2. 启动前端开发服务器
npm run dev
# 访问 http://localhost:3000

# 3. 安装后端依赖
cd ../backend
npm install

# 4. 启动后端服务
npm run dev
# 服务运行在 http://localhost:8001
```

### 常见问题

**Q: Cesium 地图不显示？**
```bash
# 检查控制台是否有 Worker 加载错误
# 确保 vite.config.js 中配置了 CESIUM_BASE_URL

# 如果仍有问题，尝试清除缓存
rm -rf node_modules/.vite
npm run dev
```

**Q: 天地图底图不加载？**
```bash
# 检查是否正确配置了天地图 API Key
# 访问 https://console.tianditu.gov.cn/ 申请 Key
```

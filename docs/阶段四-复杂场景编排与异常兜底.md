# 阶段四：复杂场景编排与异常兜底

## 4.1 多步推理业务流

### 设计原理

```
单步推理 (阶段一~三):
  用户输入 → Agent 调用一个工具 → 返回结果

多步推理 (阶段四):
  用户输入 → Agent 调用工具 A → 获取结果
                    ↓
            Agent 分析结果 → 调用工具 B → 获取结果
                    ↓
            Agent 分析结果 → 调用工具 C → 返回最终结果
```

### 实现方案：工具链 (Tool Chain)

```typescript
// 工具链定义
interface ToolChain {
  steps: ToolStep[]
  context: ChainContext
}

interface ToolStep {
  name: string
  arguments: Record<string, any>
  dependsOn?: string  // 依赖的上一步结果
}

interface ChainContext {
  // 存储每一步的执行结果
  results: Map<string, any>
  // 用户原始意图
  userIntent: string
  // 当前步骤索引
  currentStep: number
}
```

### 多步推理流程示例

```
用户输入: "查询最近的医院，并绘制 500 米范围的围栏"

Step 1: search_poi({ keyword: "医院" })
        → 返回 5 家医院列表

Step 2: 计算距离，选择最近的医院
        → 医院 A (距离 200m)

Step 3: fly_to_location({ longitude: 116.4, latitude: 39.9 })
        → 飞行到医院 A

Step 4: draw_marker({ longitude: 116.4, latitude: 39.9, name: "医院 A" })
        → 标记医院位置

Step 5: draw_circle({ longitude: 116.4, latitude: 39.9, radius: 500 })
        → 绘制 500m 围栏

Step 6: 生成自然语言回复
        → "已为你找到最近的医院 A，并绘制了 500 米范围的围栏。"
```

### 兜底回复生成

```javascript
/**
 * 生成兜底回复
 * @param {ErrorScenario} scenario - 错误场景
 * @param {Object} context - 上下文信息
 * @returns {string} 人性化回复
 */
function generateFallbackResponse(scenario, context = {}) {
  const templates = fallbackResponses[scenario]
  const template = templates[Math.floor(Math.random() * templates.length)]

  // 根据上下文定制回复
  let response = template

  if (context.keyword) {
    response += `\n\n你搜索的关键词是"${context.keyword}"，`
    response += '建议尝试更换为更通用的关键词。'
  }

  if (context.radius) {
    response += `\n\n当前搜索半径为 ${context.radius} 米，`
    response += '建议扩大到更大的范围。'
  }

  return response
}

// 使用示例
const response = generateFallbackResponse(ErrorScenario.NO_DATA, {
  keyword: '星巴克',
  radius: 500
})
// 返回: "抱歉，在当前范围内没有找到符合条件的结果。
//        你搜索的关键词是"星巴克"，建议尝试更换为更通用的关键词。
//        当前搜索半径为 500 米，建议扩大到更大的范围。"
```

## 4.4 渲染性能保护

### 数据阈值控制

```javascript
// 性能配置
const PERFORMANCE_CONFIG = {
  // POI 数量阈值
  POI_THRESHOLD: {
    LOW: 50,      // 低：全部渲染
    MEDIUM: 200,  // 中：开启聚合
    HIGH: 500     // 高：采样显示
  },

  // 渲染模式
  RENDER_MODE: {
    FULL: 'full',       // 完整渲染
    CLUSTER: 'cluster', // 聚合模式
    SAMPLE: 'sample'    // 采样模式
  }
}

/**
 * 根据数据量选择渲染模式
 * @param {number} count - POI 数量
 * @returns {string} 渲染模式
 */
function selectRenderMode(count) {
  if (count <= PERFORMANCE_CONFIG.POI_THRESHOLD.LOW) {
    return PERFORMANCE_CONFIG.RENDER_MODE.FULL
  } else if (count <= PERFORMANCE_CONFIG.POI_THRESHOLD.MEDIUM) {
    return PERFORMANCE_CONFIG.RENDER_MODE.CLUSTER
  } else {
    return PERFORMANCE_CONFIG.RENDER_MODE.SAMPLE
  }
}
```

### 聚合显示 (Clustering)

```javascript
// Cesium 聚合示例（使用 Primitive API）
function createClusteredPOIs(viewer, pois) {
  // 将 POI 按网格聚合
  const gridSize = 0.01  // 网格大小（度）
  const clusters = new Map()

  pois.forEach(poi => {
    const gridX = Math.floor(poi.location.longitude / gridSize)
    const gridY = Math.floor(poi.location.latitude / gridSize)
    const key = `${gridX}_${gridY}`

    if (!clusters.has(key)) {
      clusters.set(key, {
        center: { longitude: 0, latitude: 0 },
        count: 0,
        pois: []
      })
    }

    const cluster = clusters.get(key)
    cluster.center.longitude += poi.location.longitude
    cluster.center.latitude += poi.location.latitude
    cluster.count++
    cluster.pois.push(poi)
  })

  // 计算聚合中心点
  clusters.forEach(cluster => {
    cluster.center.longitude /= cluster.count
    cluster.center.latitude /= cluster.count
  })

  return clusters
}
```

/**
 * 数据采样
 * @param {Array} data - 原始数据
 * @param {number} maxCount - 最大数量
 * @returns {Array} 采样后的数据
 */
function sampleData(data, maxCount = 100) {
  if (data.length <= maxCount) return data

  const step = Math.ceil(data.length / maxCount)
  const sampled = []

  for (let i = 0; i < data.length; i += step) {
    sampled.push(data[i])
  }

  return sampled
}

## 4.5 完整工具链执行器

### 改进的 ToolExecutor

```typescript
// 多步工具链执行器
class ToolChainExecutor {
  private viewer: Cesium.Viewer
  private toolRegistry: Map<string, Function>
  private maxSteps = 10  // 最大步骤数

  constructor(viewer, toolRegistry) {
    this.viewer = viewer
    this.toolRegistry = toolRegistry
  }

  /**
   * 执行工具链
   * @param {ToolCall[]} chain - 工具调用链
   * @returns {Promise<ToolResult[]>} 执行结果数组
   */
  async executeChain(chain: ToolCall[]): Promise<ToolResult[]> {
    const results: ToolResult[] = []
    const context: Record<string, any> = {}

    for (let i = 0; i < chain.length && i < this.maxSteps; i++) {
      const toolCall = chain[i]

      // 替换参数中的上下文引用
      const resolvedArgs = this.resolveArguments(toolCall.arguments, context)

      // 执行工具
      const result = await this.executeTool(toolCall.name, resolvedArgs)

      // 保存结果到上下文
      context[`step_${i}`] = result
      context.lastResult = result

      results.push({
        step: i,
        toolName: toolCall.name,
        arguments: resolvedArgs,
        result
      })
    }

    return results
  }

  /**
   * 解析参数中的上下文引用
   */
  private resolveArguments(args, context) {
    const resolved = {}

    for (const [key, value] of Object.entries(args)) {
      if (typeof value === 'string' && value.startsWith('${')) {
        // 解析上下文引用
        const ref = value.slice(2, -1)
        resolved[key] = this.getNestedValue(context, ref)
      } else {
        resolved[key] = value
      }
    }

    return resolved
  }

  /**
   * 获取嵌套对象的值
   */
  private getNestedValue(obj, path) {
    return path.split('.').reduce((current, key) => current?.[key], obj)
  }

  /**
   * 执行单个工具
   */
  private async executeTool(name, args) {
    const handler = this.toolRegistry.get(name)
    if (!handler) {
      throw new Error(`未知工具: ${name}`)
    }

    return await handler(args, this.viewer)
  }
}
```
